Symfony 5.0 released November 21, 2019, same day as 4.4. Where 4.4 is stability and a long support window, 5.0 is the next chapter — no deprecated code, PHP 7.2.5 minimum, and a handful of new components that address gaps that had accumulated for years.

## :abc: The String component

PHP's string handling is famously scattered. Functions that are sometimes prefix-style (`str_`), sometimes suffix-style (`strpos`), inconsistent with encoding, and completely procedural. The String component wraps this into a fluent, unicode-aware object API:

```php
use Symfony\Component\String\UnicodeString;

$str = new UnicodeString('  Hello World  ');
echo $str->trim()->lower()->replace(' ', '-'); // hello-world
```

The practical addition is the `Slugger` — a locale-aware slug generator that handles accented characters correctly:

```php
$slug = $slugger->slug('L\'été à Montréal'); // l-ete-a-montreal
```

Previously this required either custom code or a third-party library. The slugger is now part of FrameworkBundle and available by default.

## :bell: Notifier

Email is handled by Mailer. SMS, push notifications, and chat messages had no first-party story. The Notifier component adds one: a unified interface over dozens of channels and providers.

The same notification can go to Slack, SMS via Twilio, a push notification, or a chat message, configured entirely through DSNs. Adding a new channel is a DSN configuration change, not a code change.

## :key: Secrets vault

Storing secrets in `.env` files works but has limits: the values are plain text, shared environments need secrets management, and there's no native way to encrypt them at rest.

Symfony 5.0 adds a `secrets:` command family and a vault mechanism. Secrets are encrypted with a key pair stored outside the repository. The encrypted secrets are committed; the decrypt key is not. In production, the key is an environment variable or injected from a secret manager.

```bash
php bin/console secrets:set DATABASE_PASSWORD
php bin/console secrets:decrypt-to-local --force
```

Not a full secrets management solution, but a significant improvement over unencrypted `.env` files.

## Mailer gets a notification layer

The Mailer component arrived in 4.4. What 5.0 adds on top is the `NotificationEmail` — a pre-styled, responsive email built on Foundation for Emails, with an explicit API for importance levels and call-to-action buttons:

```php
use Symfony\Bridge\Twig\Mime\NotificationEmail;

$email = (new NotificationEmail())
    ->from('alerts@example.com')
    ->to('admin@example.com')
    ->subject('Disk usage critical')
    ->markdown('The disk on **prod-01** is at 94%. Check it now.')
    ->action('Open dashboard', 'https://example.com/servers')
    ->importance(NotificationEmail::IMPORTANCE_URGENT);
```

No template to write, no inline CSS to wrestle with. For transactional alerts, billing notifications, and system emails, it covers 80% of cases out of the box.

## Lazy firewalls and the caching problem

Every stateful firewall in Symfony loads the user from session on every request — whether the action needs it or not. That makes any response uncacheable by default, even for pages that never touch `$this->getUser()`.

5.0 introduces `lazy` mode for firewalls, which defers session access until the application actually calls `is_granted()` or accesses the user token:

```yaml
# config/packages/security.yaml
security:
    firewalls:
        main:
            pattern: ^/
            anonymous: lazy
```

Pages that don't access the user become cacheable again. New Symfony projects get this by default via the Flex recipe; existing ones need a one-line config change.

## Password migrations without the big bang

Migrating a live application from bcrypt to argon2id used to mean a forced reset for all users. The `PasswordUpgraderInterface` makes it gradual: at login, Symfony checks whether the stored hash was made with the current algorithm. If not, it rehashes on the spot and calls your upgrader to persist it:

```php
// src/Repository/UserRepository.php
class UserRepository extends ServiceEntityRepository implements PasswordUpgraderInterface
{
    public function upgradePassword(UserInterface $user, string $newHashedPassword): void
    {
        $user->setPassword($newHashedPassword);
        $this->getEntityManager()->flush();
    }
}
```

Combined with `algorithm: auto` in the encoder config, old hashes silently migrate as users log in. No migration script, no downtime, no user friction.

## ErrorHandler replaces Debug

The Debug component is gone. Its replacement, ErrorHandler, does the same job — converting PHP errors to exceptions, displaying nice error pages — but without requiring Twig. For API applications that never render HTML, this matters: ErrorHandler generates errors in the format of the request (JSON, XML, plain text) following RFC 7807:

```json
{
    "title": "Not Found",
    "status": 404,
    "detail": "Sorry, the page you are looking for could not be found"
}
```

The routing config moves from `TwigBundle` to `FrameworkBundle`, and that's the only migration step for most projects.

## Event listeners, finally less verbose

Registering a kernel event listener used to require naming the event explicitly in the service tag. Symfony 5.0 infers it from the method signature:

```php
// No tag configuration needed beyond kernel.event_listener
final class SecurityListener
{
    public function onKernelRequest(RequestEvent $event): void
    {
        // Symfony reads the type hint and figures out the event
    }
}
```

```yaml
# config/services.yaml
App\EventListener\SecurityListener:
    tags: [kernel.event_listener]
```

Or use `__invoke()` and the detection is the same. Bulk-register a whole directory of listeners with one resource block, and Symfony sorts out which event each one handles.

## HttpClient grows up

The HttpClient component arrived in 4.4 as stable. 5.0 ships several additions on top of the 4.4 base that landed during the beta:

NTLM authentication for corporate environments, conditional buffering via a callback (buffer large responses only if the content-type matches), a `max_duration` option that caps the total time for a request regardless of network conditions, and `toStream()` to convert any response into a standard PHP stream for code that expects `fread()`:

```php
$response = $client->request('GET', 'https://api.example.com/large-export', [
    'max_duration' => 30.0,
    'buffer' => fn(array $headers): bool => str_contains($headers['content-type'][0] ?? '', 'json'),
]);

// Stream it instead of loading it all into memory
$stream = $response->toStream();
```

The client also gained full interoperability with PSR-18 and HTTPlug v1/v2, so any library that depends on those abstractions works with it directly.

## :broom: What 5.0 removes

5.0 drops everything deprecated in 4.4. The most notable:
- `WebServerBundle` (use `symfony server:start` from the CLI tool instead)
- The old security system's `AnonymousToken` (replaced by `NullToken`)
- Old form event names
- Symfony's internal ClassLoader
- The Debug component (replaced by ErrorHandler)

If you ran your 4.4 application with deprecation notices active and fixed the warnings, 5.0 requires no code changes.
