PHP 7.4 landed November 28th. It's the last 7.x release before PHP 8.0, and it feels like it — the features are substantial enough to stand on their own, but they also read as groundwork for what's coming.

## :label: Typed properties

This is the one. Since PHP 7.0, you could type function parameters and return values. But class properties? Still untyped:

```php
class User {
    public int $id;
    public string $name;
    public ?DateTimeInterface $deletedAt;
}
```

7.4 changes that. Typed properties enforce types at assignment, not just at call sites. The practical effect: classes become self-documenting in a way that docblocks never quite managed, and the engine catches type errors before they propagate.

One subtlety: typed properties are `uninitialized` by default (not `null`). Accessing an uninitialized property throws an `Error`. This surprises people who assume `?string` implies a default of `null` — it doesn't. You still need an explicit `= null` for that.

## :arrow_right: Arrow functions

Closures in PHP have always required explicitly importing outer scope variables with `use`:

```php
$multiplier = 3;
$fn = fn($x) => $x * $multiplier; // no use() needed
```

Arrow functions capture the enclosing scope automatically. Single expression, implicit return, no boilerplate. They don't replace full closures for complex logic, but for the common case of a short callback they eliminate a class of noise that accumulated over years.

## :crystal_ball: Opcache preloading

For long-lived PHP-FPM setups, preloading allows a script to load and compile PHP files into opcache memory at server startup. Those files are available to all requests without compilation overhead.

The performance gain varies by application. On large frameworks where the same files are loaded on every request, the improvement is real. On smaller apps, negligible. Worth benchmarking before adding the configuration complexity.

## :comet: What comes next

PHP 7.4 includes a `??=` assignment operator, spread operator in arrays, and covariant return types — each small but useful. The real anticipation is for 8.0. The RFC backlog is substantial. 7.4 is a good place to land while waiting.
