PHP 8.1 released November 25th. It follows 8.0's sweeping overhaul with something different: a smaller set of features, each executed with more depth than a surface-level addition.

## :card_index_dividers: Enums

This is the one that changes codebases immediately. Before 8.1, PHP enumerations were either class constants, strings, or integers with no enforcement:

```php
// before: nothing stops Status::INVALID from being passed
const ACTIVE = 'active';
const INACTIVE = 'inactive';

// after
enum Status: string {
    case Active = 'active';
    case Inactive = 'inactive';
}

function activate(Status $status): void { ... }
```

PHP enums are objects, not scalars. They support methods, interfaces, and constants. Backed enums (with a string or int value) serialize cleanly and map to database columns naturally. Pure enums (no backing type) enforce domain concepts without serialization concerns.

The immediate effect: every status field, every finite set of states in every codebase I maintain became an enum candidate. The type system finally has a native construct for the thing every PHP project has been approximating for years.

## :thread: Fibers

Fibers are a cooperative concurrency primitive: a way to pause and resume execution of a function, yielding control without threads.

```php
$fiber = new Fiber(function(): void {
    $value = Fiber::suspend('first');
    echo "Resumed with: {$value}\n";
});

$result = $fiber->start();    // 'first'
$fiber->resume('hello');      // "Resumed with: hello"
```

Fibers are the foundation async libraries like ReactPHP and Amp needed from the runtime itself. For most application developers the direct API matters less than the libraries that build on top of it — but understanding fibers explains what those libraries are doing underneath.

## :pencil2: Readonly properties

8.0 brought constructor promotion. 8.1 adds `readonly`:

```php
class User {
    public function __construct(
        public readonly int $id,
        public readonly string $name,
    ) {}
}
```

A `readonly` property can be written exactly once — during initialization. After that, any write throws an `Error`. Combined with constructor promotion, value objects and DTOs become concise and semantically correct.

## :memo: First-class callable syntax

```php
$fn = strlen(...);
$fn = $this->process(...);
$fn = MyClass::create(...);
```

`...` after a callable creates a `Closure` without `Closure::fromCallable()` boilerplate. Useful when passing methods as callbacks.

8.1 is precise. Enums alone justify the upgrade.
