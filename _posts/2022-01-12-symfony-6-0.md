Symfony 6.0 released November 29, 2021. The defining characteristic: PHP 8.1 is the minimum. Not supported — required. The releases team waited for PHP 8.1 to ship, then cut Symfony 6.0 the next day.

This isn't just a version bump. It's a commitment to build against the modern language rather than the historical floor.

## :closed_lock_with_key: The security system, finally rebuilt

The Symfony security component has two systems. The old one — `AnonymousToken`, `GuardAuthenticatorInterface`, a tangle of interfaces that required implementing methods you didn't need — had been marked deprecated. 6.0 removes it entirely.

The new security system (`security.enable_authenticator_manager: true` in 5.x) is now the only system. It's cleaner: one interface to implement, clear separation between authentication and authorization, passport-based approach for credential checking. The upgrade from the old guard authenticators is not trivial, but the destination is significantly less confusing.

## :file_folder: The Filesystem Path class

Working with filesystem paths in PHP is a string manipulation problem. `__DIR__`, concatenation, `realpath()`, platform-specific separators — the standard library gives you primitives but no model.

The new `Path` class handles this:

```php
use Symfony\Component\Filesystem\Path;

Path::join('/var/www', 'html', '../uploads'); // /var/www/uploads
Path::makeRelative('/var/www/html', '/var/www'); // html
Path::isAbsolute('./relative/path'); // false
```

Cross-platform, no side effects, no filesystem access required. Also in 6.0: nested `.gitignore` pattern support in Finder.

## :card_index_dividers: Enums in the form system

Following 5.4's groundwork, 6.0's enum support extends further. `BackedEnum` values round-trip through forms and the serializer without custom transformers. The form component understands enum cases as choice options automatically.

## :wastebasket: What 6.0 removes

The removal list is extensive: the old security system, the `Templating` component, PHP annotations support (replaced by native attributes), Doctrine Cache support, `ContainerAwareTrait`. Six years of accumulated `@deprecated` markers cleaned out.

Applications that treated 5.4 deprecation warnings seriously had a clean upgrade path. Applications that didn't had work to do.
