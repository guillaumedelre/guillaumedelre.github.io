Symfony 6.4 landed November 29, 2023. It's an LTS with a story: four components that shipped as experimental in earlier releases are now stable. The most consequential is AssetMapper.

## :package: AssetMapper

Modern frontend tooling in Symfony meant Webpack Encore. Encore works — it handles transpilation, bundling, versioning, hot reload. It also requires Node.js, a separate build step, and a meaningful amount of configuration for what is often a modest frontend.

AssetMapper takes a different position: modern browsers support ES modules natively. Rather than bundling, ship the files as-is, let the browser resolve imports through an importmap, and manage vendor dependencies through downloaded files rather than npm packages.

```bash
composer require symfony/asset-mapper
php bin/console importmap:require lodash
```

No Node.js. No npm. No build step. JavaScript and CSS files are versioned and served directly, with a digest in the URL for cache busting. For applications where the frontend is not the primary engineering concern, this removes an entire toolchain.

6.4 adds CSS files to the importmap, automatic CSS preloading via WebLink, and commands to audit and update vendor dependencies — the package.json experience, but without npm.

## :calendar: Scheduler

The Scheduler component — periodic and cron-style task scheduling without an external job runner — exits experimental and becomes stable. The API uses attributes:

```php
#[AsCronTask('0 * * * *')]
class HourlyReport implements ScheduledTaskInterface
{
    public function run(): void { ... }
}
```

Backed by Messenger transports, tasks can run in any environment where a worker is running. The scheduler replaces the `cron` entry + artisan-style command pattern for many use cases.

## :hook: Webhook and RemoteEvent

Also graduating from experimental: the Webhook component handles incoming webhooks from external services. Rather than writing raw controllers that parse payloads and dispatch events, you configure parsers for known services (Stripe, GitHub, Mailgun) and receive typed events.

## :clock3: DatePoint

A new `DatePoint` class in the Clock component: an immutable `DateTime` wrapper that throws exceptions on invalid modifiers rather than silently returning `false`. Small but meaningful for code that manipulates dates and expects to know when something goes wrong.

## :calendar: The support window

6.4 LTS receives bug fixes until November 2026 and security fixes until November 2027. The path from 6.4 to 7.4 (the next LTS) runs through the 6.4 deprecation notices.
