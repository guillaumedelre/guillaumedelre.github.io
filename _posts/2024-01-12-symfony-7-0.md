Symfony 7.0 released November 29, 2023 ‚Äî same day as 6.4. The pattern holds: the X.0 release cuts deprecated code and raises the PHP minimum. 7.0 requires PHP 8.2 and removes everything that 6.4 flagged.

The most visible removal: Doctrine annotations. `@Route`, `@ORM\Column`, `@Assert` ‚Äî gone. Native PHP attributes have been the recommended approach since Symfony 5.2. 7.0 makes it mandatory.

## :label: Attributes everywhere

The migration from annotations to attributes is mostly mechanical ‚Äî syntax changes from `@` to `#[]`, and the class references change from Doctrine annotation classes to PHP attribute classes:

```php
// before
/** @Route('/users', methods={"GET"}) */

// after
#[Route('/users', methods: ['GET'])]
```

The payoff beyond syntax: attributes are validated by the PHP engine, not a docblock parser. IDEs can resolve them without custom plugins. Static analysis tools understand them natively.

## :workflow: Workflow with PHP attributes

Workflow event listeners and guards can now be registered via attributes:

```php
#[AsGuard(workflow: 'order', transition: 'ship')]
public function canShip(Event $event): void
{
    if (!$event->getSubject()->isPaymentConfirmed()) {
        $event->setBlocked(true);
    }
}
```

The workflow profiler ‚Äî a dedicated panel showing the current marking and available transitions ‚Äî exits as a useful debugging tool.

## :clock1: DatePoint in the Clock component

`DatePoint` ‚Äî the immutable `DateTime` with strict error handling introduced in 6.4 ‚Äî is now the recommended way to create and manipulate dates. Combined with PHP 8.2's readonly properties, date value objects in domain code become straightforward:

```php
readonly class Order {
    public function __construct(
        public DatePoint $createdAt,
        public ?DatePoint $shippedAt = null,
    ) {}
}
```

## :wastebasket: What 7.0 removes

The full removal list includes: Doctrine annotations support, the `Templating` component bridge, `ProxyManager` bridge, the `Monolog` bridge for versions below 3.0, and Sendinblue transport (replaced by Brevo). PHP 8.0 and 8.1 support also ends ‚Äî 8.2 is the floor.

Upgrade from 6.4 with all deprecation notices fixed, and 7.0 is straightforward. Skip that step and it isn't.

## Scheduler and AssetMapper graduate

Two components that shipped as experimental in 6.3 are now stable: Scheduler and AssetMapper. Stable means their APIs are locked, no more `@experimental` caveats, and they appear in the upgrade guide.

Scheduler gets `#[AsCronTask]` and `#[AsPeriodicTask]` for attribute-based task registration, runtime schedule modification with heap recalculation, `FailureEvent`, and a `--date` option on `schedule:debug`. AssetMapper adds CSS file support in importmap, an `outdated` command, an `audit` command, and automatic preloading via WebLink.

```php
#[AsCronTask('0 2 * * *')]
class NightlyReportMessage {}

#[AsPeriodicTask(frequency: '1 hour')]
class HourlyCleanupMessage {}
```

## Service wiring gets two new attributes

`#[AutowireLocator]` and `#[AutowireIterator]` landed in 6.4 and are non-experimental in 7.0. They replace the XML/YAML tagged service locator configuration with inline PHP:

```php
class HandlerRegistry
{
    public function __construct(
        #[AutowireLocator('app.handler', indexAttribute: 'key')]
        private ContainerInterface $handlers,
    ) {}
}
```

The `#[Target]` attribute also gets smarter: when a service has a named autowiring alias like `invoice.lock.factory`, you can now write `#[Target('invoice')]` instead of the full alias name. Less noise when the type already implies what you want.

## Messenger gets more precise failure handling

`RejectRedeliveredMessageException` tells the worker not to retry a message ‚Äî useful when a message arrives twice due to a transport ack timeout and you want exactly-once semantics. The `messenger:failed:remove --all` option clears the entire failure transport without scripting a loop. Failed retries can also be resent directly to the failure transport, bypassing the retry queue.

Multiple Redis Sentinel hosts are now supported in the DSN:

```
redis-sentinel://host1:26379,host2:26379,host3:26379/mymaster
```

## Console gets signal names and command profiling

`SignalMap` maps signal integers to their POSIX names. When a worker catches `SIGTERM`, the log now says `SIGTERM` instead of `15`. `ConsoleTerminateEvent` is dispatched even when the process exits due to a signal ‚Äî before 7.0, a signal exit skipped the termination event entirely.

Command profiling lands too: running `bin/console` with `--profile` stores the collected profile in the Symfony profiler, browsable via the web interface.

## Form: small things that add up

`ChoiceType` gets a `duplicate_preferred_choices` option ‚Äî set it to `false` to avoid showing the same option twice when preferred choices overlap with the full list. `FormEvent::setData()` is deprecated for events where changing data has no effect (the data is already locked by that point in the lifecycle). The self-closing slash on `<input>` elements is removed ‚Äî `<input>` is a void element in HTML5, the slash was technically invalid.

Enum support in forms arrives: `ChoiceType` renders backed enums directly, and translatable enums render their labels through the translator without custom configuration.

## HttpFoundation: small but useful

`Response::send()` gets a `$flush` parameter. Pass `false` to buffer the output without flushing to the client ‚Äî useful when chaining middleware that inspects the response before it leaves the process.

`UriSigner` moves from HttpKernel to HttpFoundation, where it belongs semantically. The class name stays the same, only the namespace changes.

Cookies get CHIPS support (Cookies Having Independent Partitioned State), the browser mechanism for cross-site cookies in a first-party partition. Relevant if you build embeddable widgets.

## Translation: Phrase provider and tree output

Phrase joins Crowdin and Lokalise as a supported translation provider. Configure it in `config/packages/translation.yaml` and `translation:push` / `translation:pull` handle the sync.

`translation:pull` gets an `--as-tree` option that writes translation files in nested YAML rather than flat dot-notation keys. Whether that's better depends on your team.

`LocaleSwitcher::runWithLocale()` now passes the current locale as an argument to the callback, which saves a `getLocale()` call inside:

```php
$switcher->runWithLocale('fr', function (string $locale) use ($mailer) {
    $mailer->send($this->buildEmail($locale));
});
```

## A few things in Serializer and DomCrawler

The Serializer's `Context` attribute can now target specific classes, so a single DTO can have different (de)serialization behavior depending on which class holds the context. `TranslatableNormalizer` arrives for normalizing objects that implement `TranslatableInterface` ‚Äî the translator is called during normalization, not before.

`Crawler::attr()` gains a `$default` parameter. Instead of null-checking the return value, pass a fallback:

```php
$src = $crawler->attr('src', '/placeholder.png');
```

`assertAnySelectorText()` and `assertAnySelectorTextContains()` join the DomCrawler assertion set ‚Äî they pass if at least one matching element satisfies the condition, rather than requiring all of them to.

## HttpClient: HAR responses for testing

`MockResponse` accepts HAR (HTTP Archive) files. Record real HTTP interactions in your browser or with a proxy, drop the `.har` file in your test fixtures, and replay them:

```php
$client = new MockHttpClient(HarFileResponseFactory::createFromFile(__DIR__.'/fixtures/api.har'));
```

Beats writing response stubs by hand for complex API interactions.
