Symfony 7.0 released November 29, 2023 — same day as 6.4. The pattern holds: the X.0 release cuts deprecated code and raises the PHP minimum. 7.0 requires PHP 8.2 and removes everything that 6.4 flagged.

The most visible removal: Doctrine annotations. `@Route`, `@ORM\Column`, `@Assert` — gone. Native PHP attributes have been the recommended approach since Symfony 5.2. 7.0 makes it mandatory.

## :label: Attributes everywhere

The migration from annotations to attributes is mostly mechanical — syntax changes from `@` to `#[]`, and the class references change from Doctrine annotation classes to PHP attribute classes:

```php
// before
/** @Route('/users', methods={"GET"}) */

// after
#[Route('/users', methods: ['GET'])]
```

The payoff beyond syntax: attributes are validated by the PHP engine, not a docblock parser. IDEs can resolve them without custom plugins. Static analysis tools understand them natively.

## :workflow: Workflow with PHP attributes

Workflow event listeners and guards can now be registered via attributes:

```php
#[AsGuard(workflow: 'order', transition: 'ship')]
public function canShip(Event $event): void
{
    if (!$event->getSubject()->isPaymentConfirmed()) {
        $event->setBlocked(true);
    }
}
```

The workflow profiler — a dedicated panel showing the current marking and available transitions — exits as a useful debugging tool.

## :clock1: DatePoint in the Clock component

`DatePoint` — the immutable `DateTime` with strict error handling introduced in 6.4 — is now the recommended way to create and manipulate dates. Combined with PHP 8.2's readonly properties, date value objects in domain code become straightforward:

```php
readonly class Order {
    public function __construct(
        public DatePoint $createdAt,
        public ?DatePoint $shippedAt = null,
    ) {}
}
```

## :wastebasket: What 7.0 removes

The full removal list includes: Doctrine annotations support, the `Templating` component bridge, `ProxyManager` bridge, the `Monolog` bridge for versions below 3.0, and Sendinblue transport (replaced by Brevo). PHP 8.0 and 8.1 support also ends — 8.2 is the floor.

Upgrade from 6.4 with all deprecation notices fixed, and 7.0 is straightforward. Skip that step and it isn't.
