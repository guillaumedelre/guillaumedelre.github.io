Symfony 7.4 released November 2025, alongside 8.0. It's the last LTS of the 7.x line — PHP 8.2 minimum, three years of bug fixes, four of security. For teams that can't or won't follow the PHP 8.4 requirement that 8.0 demands, 7.4 is the destination.

## :envelope_with_arrow: Message signing in Messenger

Transport security in Messenger has always been the application's responsibility. 7.4 adds message signing: a stamp-based mechanism that signs dispatched messages and validates signatures on reception.

The use case is multi-tenant or external transport scenarios where you need cryptographic proof that a message wasn't tampered with or injected from outside. Configuration is transport-level:

```yaml
framework:
    messenger:
        transports:
            async:
                dsn: '%env(MESSENGER_TRANSPORT_DSN)%'
                options:
                    signing_key: '%env(MESSENGER_SIGNING_KEY)%'
```

## :gear: PHP array configuration

Symfony's configuration formats have always been YAML (default), XML, and PHP. The PHP format existed but was awkward: a fluent builder DSL that required method chaining and offered no autocompletion.

7.4 replaces the fluent format with standard PHP arrays. The result is configuration that IDEs can analyze, `config/reference.php` is auto-generated as a type-annotated reference, and the format reads like data rather than code:

```php
return static function (FrameworkConfig $framework): void {
    $framework->router()->strictRequirements(null);
    $framework->session()->enabled(true);
};
```

The fluent format is deprecated; arrays are the future.

## :shield: OIDC improvements

The `#[IsSignatureValid]` attribute validates signed URLs directly in controllers, removing the boilerplate of manual validation. OpenID Connect gets support for multiple discovery endpoints, and a new `security:oidc-token:generate` command helps with development and testing.

## :calendar: The support window

7.4 LTS: bugs until November 2028, security fixes until November 2029. The bridge to 8.4 LTS (the next long-term target) runs through 7.4's deprecation notices and the PHP 8.4 upgrade.

## Attributes get more precise

`#[CurrentUser]` now accepts union types, which matters when a route can be reached by more than one user class:

```php
public function index(#[CurrentUser] AdminUser|Customer $user): Response
```

`#[Route]` accepts an array for the `env` option. A debug route active only in `dev` and `test`, or a health check active in `staging` and `prod`, no longer needs two separate route definitions. `#[AsDecorator]` becomes repeatable, so one class can decorate multiple services at once. `#[AsEventListener]` method signatures accept union event types. `#[IsGranted]` gets a `methods` option to scope an authorization check to specific HTTP verbs without duplicating the route.

## Request class stops doing too much

`Request::get()` is deprecated. The method searched route attributes, then query parameters, then request body, in that order, silently returning whatever it found first. The ambiguity caused real bugs. In 8.0 it's gone; in 7.4 it still works but triggers a deprecation. The replacements are explicit: `$request->attributes->get()`, `$request->query->get()`, `$request->request->get()`.

Body parsing for `PUT`, `PATCH`, `DELETE`, and `QUERY` requests arrives at the same time. Previously Symfony only parsed `application/x-www-form-urlencoded` and `multipart/form-data` for `POST`. Now those same content types are parsed for the other writable methods too, which removes a common REST API workaround.

HTTP method override for `GET`, `HEAD`, `CONNECT`, and `TRACE` is deprecated. Overriding a safe method with a header always was semantically broken. You can now explicitly allow only the methods that make sense for your app:

```php
Request::setAllowedHttpMethodOverride(['PUT', 'PATCH', 'DELETE']);
```

## Workflows accept BackedEnums

Workflow places and transitions can now be defined with PHP backed enums, both in YAML (via the `!php/enum` tag) and in PHP config. The marking store works with enum values directly, so the domain model and the workflow definition use the same types:

```yaml
framework:
    workflows:
        blog_publishing:
            initial_marking: !php/enum App\Status\PostStatus::Draft
            places: !php/enum App\Status\PostStatus
            transitions:
                publish:
                    from: !php/enum App\Status\PostStatus::Review
                    to: !php/enum App\Status\PostStatus::Published
```

## Extending validation and serialization for third-party classes

If you use a class from a bundle or library and need to add validation or serialization metadata to it, 7.4 gives you `#[ExtendsValidationFor]` and `#[ExtendsSerializationFor]`. You write a companion class with your extra annotations, point the attribute at the target class, and Symfony merges the metadata at container compilation time:

```php
#[ExtendsValidationFor(UserRegistration::class)]
abstract class UserRegistrationValidation
{
    #[Assert\NotBlank(groups: ['my_app'])]
    #[Assert\Length(min: 3, groups: ['my_app'])]
    public string $name = '';

    #[Assert\Email(groups: ['my_app'])]
    public string $email = '';
}
```

Symfony verifies at compile time that the declared properties actually exist on the target class, so a rename won't silently break your validation.

## DX: the things that don't headline but matter

The Question helper in Console accepts a timeout. Ask the user to confirm something, and if they don't respond in N seconds, the default answer applies. Useful in deployment scripts that must not block forever.

`messenger:consume` gets `--exclude-receivers`. Combined with `--all`, it lets you consume from every transport except specific ones:

```bash
bin/console messenger:consume --all --exclude-receivers=low_priority
```

FrankenPHP worker mode is now auto-detected. If the process is running inside FrankenPHP, Symfony switches to worker mode automatically without the `runtime/frankenphp-symfony` package.

The `debug:router` command hides the `Scheme` and `Host` columns when all routes use `ANY`, reducing noise in the default case. HTTP methods in the output are color-coded.

Functional tests get `$client->getSession()` before the first request. Previously you had to make at least one request to access the session. Now you can pre-seed CSRF tokens or A/B testing flags:

```php
$session = $client->getSession();
$session->set('_csrf/checkout', 'test-token');
$session->save();
```

## Lock: DynamoDB store

`DynamoDbStore` lands as a new Lock backend. Useful in AWS-native deployments where Redis isn't in the stack. It works like any other store:

```php
$store = new DynamoDbStore('dynamodb://default/locks');
$factory = new LockFactory($store);
```

## Doctrine bridge: day and time point types

Two new Doctrine column types: `day_point` stores a date-only value (no time component) and `time_point` stores a time-only value, both mapping to `DatePoint`. Useful when the semantics of your domain explicitly separate date from time:

```php
#[ORM\Column(type: 'day_point')]
public DatePoint $birthDate;

#[ORM\Column(type: 'time_point')]
public DatePoint $openingTime;
```

## Routing: explicit query parameters

The `_query` key in URL generation lets you set query parameters explicitly, separate from route parameters. The difference matters when a route parameter and a query parameter share the same name:

```php
$url = $urlGenerator->generate('report', [
    'site' => 'fr',
    '_query' => ['site' => 'us'],
]);
// /report/fr?site=us
```

## WebLink: parsing incoming Link headers

`HttpHeaderParser` parses `Link` response headers into structured objects. Before this, parsing Link headers from API responses required either a third-party library or regex. The use case is HTTP APIs that advertise related resources or pagination via Link headers, as GitHub's API does.

## HTML5 parsing gets faster on PHP 8.4

DomCrawler and HtmlSanitizer switch to PHP 8.4's native HTML5 parser when available. No code changes needed. The native parser is faster and more spec-compliant than the previous fallback. On PHP 8.2 or 8.3 the behavior is unchanged.

## Translation: StaticMessage

`StaticMessage` implements `TranslatableInterface` but does not translate. It passes the string through unchanged regardless of locale. The use case is API responses that must stay in a fixed language even when the user's locale differs, or audit log entries where the original text must be preserved.
