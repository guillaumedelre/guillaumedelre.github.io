Symfony 7.4 released November 2025, alongside 8.0. It's the last LTS of the 7.x line — PHP 8.2 minimum, three years of bug fixes, four of security. For teams that can't or won't follow the PHP 8.4 requirement that 8.0 demands, 7.4 is the destination.

## :envelope_with_arrow: Message signing in Messenger

Transport security in Messenger has always been the application's responsibility. 7.4 adds message signing: a stamp-based mechanism that signs dispatched messages and validates signatures on reception.

The use case is multi-tenant or external transport scenarios where you need cryptographic proof that a message wasn't tampered with or injected from outside. Configuration is transport-level:

```yaml
framework:
    messenger:
        transports:
            async:
                dsn: '%env(MESSENGER_TRANSPORT_DSN)%'
                options:
                    signing_key: '%env(MESSENGER_SIGNING_KEY)%'
```

## :gear: PHP array configuration

Symfony's configuration formats have always been YAML (default), XML, and PHP. The PHP format existed but was awkward: a fluent builder DSL that required method chaining and offered no autocompletion.

7.4 replaces the fluent format with standard PHP arrays. The result is configuration that IDEs can analyze, `config/reference.php` is auto-generated as a type-annotated reference, and the format reads like data rather than code:

```php
return static function (FrameworkConfig $framework): void {
    $framework->router()->strictRequirements(null);
    $framework->session()->enabled(true);
};
```

The fluent format is deprecated; arrays are the future.

## :shield: OIDC improvements

The `#[IsSignatureValid]` attribute validates signed URLs directly in controllers, removing the boilerplate of manual validation. OpenID Connect gets support for multiple discovery endpoints, and a new `security:oidc-token:generate` command helps with development and testing.

## :calendar: The support window

7.4 LTS: bugs until November 2028, security fixes until November 2029. The bridge to 8.4 LTS (the next long-term target) runs through 7.4's deprecation notices and the PHP 8.4 upgrade.
